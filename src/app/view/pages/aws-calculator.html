Fase 5 — Comparativo AWS

Metas da Entrega 2

Objetivo: Usar a AWS Pricing Calculator para estimar os custos (On-Demand – 100%) de uma máquina Linux simples nas regiões:

  • São Paulo (BR)
  • Virgínia do Norte (EUA)

A máquina hospedaria uma API responsável por receber os dados dos sensores (Entrega 1) e executar o Machine Learning.

Configurações

  • 2 CPUs
  • 1 GiB RAM
  • Até 5 Gbps de rede
  • 50 GB de armazenamento

Comparação de Custos

A tabela abaixo compara os custos mensais em USD para diferentes opções de pagamento e regiões.

Região Compute Saving Plans EC2 Instance Saving Plans Sob Demanda Instâncias Spot
São Paulo 2.41 2.12 4.89 0.59
Virgínia do Norte 1.53 1.31 3.07 1.59

Conclusões

A solução mais barata para as configurações analisadas é o uso de Instâncias Spot na região de São Paulo, com custo mensal de apenas 0,59 USD.

Esse custo reduzido está relacionado ao modelo de uso de capacidade excedente dos EC2 locais.

Embora as Instâncias Spot ofereçam baixo custo, elas possuem limitações importantes, como risco de interrupção. Entretanto, como não foi mencionado que o projeto exige alta tolerância a falhas, essa opção é viável para uma versão inicial, permitindo escalar conforme a demanda.

Restrições legais e latência

Considerando que existe restrição para armazenamento de dados no exterior e que você precisa acessar rapidamente os dados coletados pelos sensores:

A escolha mais adequada seria a região de São Paulo.

  • Atende as restrições legais de armazenamento
  • Menor latência para acessar os sensores
  • Competitiva em custos na modalidade Instâncias Spot
